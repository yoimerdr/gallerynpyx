Quickstart
==========

This guide shows how to integrate gallerynpyx into a Ren'Py project using the
``tutorial`` sample assets. It assumes you already know the basics of Ren'Py.
If not, the official
`quickstart <https://www.renpy.org/doc/html/quickstart.html>`_
is a good place to begin.

gallerynpyx 2.0.0 keeps the shared gallery flow from 1.x, but also adds named local
galleries. Each local gallery can keep its own handler state plus its own resources,
screens and styles.

Install gallerynpyx
-------------------

Download the
`latest release <https://www.github.com/yoimerdr/gallerynpyx/releases/tag/gallerynpyx-latest>`_
and extract the ``gallerynpyx`` files into the game's ``game`` folder.

Shared gallery
--------------

The shared gallery is still the default entry point in 2.0.0. It uses the unnamed
handler returned by :func:`gallerynpyx.get_handler` together with the shared
configuration objects from :mod:`gallerynpyx.config`.

.. code-block:: renpy

    image gx_image = "images/concert1.png"

    image gx_animation:
        "concert1" with Dissolve(.1)
        pause .4
        "concert2" with Dissolve(.1)
        pause .4
        "concert3" with Dissolve(.1)
        pause .4
        "concert2" with Dissolve(.1)
        pause .4
        repeat

    init python:
        gx_resources = gallerynpyx.config.get_resources()
        gx_resources.allow_animation_speeds = True
        gx_resources.allow_animation_thumbnail = True
        gx_resources.transition = fade

        gx_screens = gallerynpyx.config.get_screens()
        gx_screens.background = "#10141d"
        gx_screens.show_scrollbar = True

        gx_styles = gallerynpyx.config.get_styles()
        gx_styles.navigation = "gx_navigation"
        gx_styles.controls = "gx_controls"
        gx_styles.tooltip = "gx_tooltip"

        gx_handler = gallerynpyx.get_handler()
        gx_handler.put("images", "gx_image")

        gx_handler.put(
            "animations",
            gallerynpyx.resources.AnimationResource("gx_animation"),
        )

        gx_handler.put("videos", "oa4_launch.webm", thumbnail="concert1")
        gx_handler.to_first()

To open the built-in gallery screen from your UI:

.. code-block:: renpy

    screen navigation():
        vbox:
            textbutton _("Gallery") action ShowMenu("gallerynpyx")

Named local galleries
---------------------

gallerynpyx 2.0.0 can also create named local galleries. A local gallery keeps its own
handler state and can resolve gallery-specific resources, screens and styles.

.. code-block:: renpy

    init python:
        gx_extras_handler = gallerynpyx.get_handler("extras")

        gx_extras_resources = gallerynpyx.config.get_resources("extras")
        gx_extras_resources.allow_animation_speeds = True
        gx_extras_resources.allow_animation_thumbnail = True
        gx_extras_resources.transition = dissolve

        gx_extras_screens = gx_extras_handler.screens_config
        gx_extras_screens.background = "#141821"
        gx_extras_screens.show_scrollbar = True

        gx_extras_styles = gx_extras_handler.styles_config
        gx_extras_styles.navigation = "gx_extras_navigation"
        gx_extras_styles.controls = "gx_extras_controls"
        gx_extras_styles.tooltip = "gx_extras_tooltip"

        gx_extras_handler.put("images", "gx_image")
        gx_extras_handler.put(
            "animations",
            gallerynpyx.resources.AnimationResource("gx_animation"),
        )
        gx_extras_handler.put("videos", "oa4_launch.webm", thumbnail="concert1")
        gx_extras_handler.to_first()

You can open that gallery by name or by passing the handler instance:

.. code-block:: renpy

    screen navigation():
        vbox:
            textbutton _("Main Gallery") action ShowMenu("gallerynpyx")
            textbutton _("Extras Gallery") action ShowMenu("gallerynpyx", gx_handler="extras")
            textbutton _("Extras Gallery (Handler)") action ShowMenu("gallerynpyx", gx_handler=gx_extras_handler)


Galleries with sliders
----------------------

Use slides and sliders when you want to group items by chapter, route, character or any
other category. ``Slide`` represents a list of gallery items, while ``Slider`` groups
slides and other sliders.

For example, :meth:`gallerynpyx.handler.Handler.put` creates the destination slide if it
does not exist yet. In the shared gallery above, that is how the slides ``images``,
``animations`` and ``videos`` are created.

To nest those slides inside a slider called ``Chapter 1``:

.. code-block:: renpy

    init python:
        gx_handler = gallerynpyx.get_handler()
        chapter1 = gallerynpyx.create_slider("ch1", label="Chapter 1")
        gx_handler.root.add(chapter1)

        gx_handler.put(("ch1", "images"), "gx_image")
        gx_handler.put(
            ("ch1", "animations"),
            gallerynpyx.resources.AnimationResource("gx_animation"),
        )
        gx_handler.put(("ch1", "videos"), "oa4_launch.webm", thumbnail="concert1")

Custom screens in 2.0.0
-----------------------

The built-in gallery screens now work with whichever handler is passed in through
``gx_handler``. If you replace any built-in screen, keep that parameter in the screen
signature and resolve it before using actions or configs.

.. code-block:: renpy

    screen gx_controls(has_animations=False, gx_handler=None):
        $ gx_handler = gallerynpyx.handler.coerce(name=gx_handler)
        $ gx_config = gx_handler.styles_config

        vbox:
            style gx_config.controls
            style_prefix gx_config.controls

            textbutton _("Previous"):
                action gallerynpyx.actions.PreviousPage(gx_handler)

            textbutton _("Next"):
                action gallerynpyx.actions.NextPage(gx_handler)

            textbutton _("Return"):
                action gallerynpyx.actions.ReturnSlide(has_animations, gx_handler)

This matters both for named local galleries and for any custom screen tree that reuses
the built-in actions.


Gallery items
-------------

Animations with speed
^^^^^^^^^^^^^^^^^^^^^

If you want to expose speed controls for animations, enable them through the resources
configuration used by the gallery.

.. code-block:: renpy

    init python:
        gx_handler.put(
            "animations",
            gallerynpyx.resources.AnimationResource("gx_animation"),
            thumbnail="concert1",
        )

        gx_config = gx_handler.resources_config
        gx_config.allow_animation_speeds = True

.. note::
    This setting affects all animations displayed through that configuration object.
    In a named local gallery, it only affects that gallery.

These controls work automatically for simple ATL sequences. In more complex animations,
make the timing depend on ``animation_speed`` explicitly:

.. code-block:: renpy

    image gx_complex_animation:
        "concert1" with Dissolve(.1)
        pause 0.4 / gx_handler.resources_config.animation_speed
        "concert2" with Dissolve(.1)
        pause 0.4 / gx_handler.resources_config.animation_speed
        repeat
