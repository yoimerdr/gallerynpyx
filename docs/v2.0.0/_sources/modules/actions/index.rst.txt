actions
=======

All functions, classes and objects defined in the submodules are available for direct
import, without the need to specify the paths individually.

In ``2.0.0``, the handler-aware actions can target either the shared gallery or a named
local gallery. They accept a handler instance or gallery name so the same custom screen
can be reused across multiple gallery scopes.
