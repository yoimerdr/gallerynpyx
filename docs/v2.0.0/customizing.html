    Customizing

    gallerynpyx 2.0.0 keeps customization centered on three configuration objects:

    • StylesConfig

    • ScreensConfig

    • ResourcesConfig

    The shared gallery uses the unnamed configuration objects, while named local galleries resolve their own instances through the handler or through gallerynpyx.config.

    Important

    If you replace one of the built-in screens, keep gx_handler=None in the screen signature and forward that value when you call other gallery screens or actions. This is what allows the same screen tree to work with both the shared gallery and named local galleries.

    Customization reference

    Main screen

    The main screen is the default view displayed when accessing the gallerynpyx screen. This screen contains some elements that you can customize, such as the background, the foreground and the root screen.

    Screen declaration:

    screen gallerynpyx(gx_handler=None, gx_size=None):
    # ...

    Background

    The background is the first visual element added to the main screen. It can be any valid resource: images, solid colors, animations, among others.

    Related properties:

    • background

    Foreground

    The foreground is the layer that overlays the background. It can also contain any valid resource.

    Related properties:

    • foreground

    Root screen

    The root screen acts as the main container of the interface. It defines the general structure and organizes the other screens or visual elements.

    This screen can be customized or replaced.

    Screen declaration:

    screen gx_root(gx_handler=None):
    # ...

    Related properties:

    • root_screen

    • root

    Tooltip screen

    The tooltip screen displays additional or contextual information about specific items. It is generally used to display brief explanations, instructions or details when hovering over an item.

    This screen can be customized or replaced.

    See section 1 in Customization reference for how and where this screen is displayed by default.

    Screen declaration:

    screen gx_tooltip(text, gx_handler=None):
    # ...

    Related properties:

    • tooltip_screen

    • tooltip

    The controls screen includes the buttons to change pages within the current slide, or to return from the slides or the main gallery.

    Items screen

    The items screen is used to display and manage the items within a specific page in a slide.

    This screen can be customized or replaced.

    See section 3 in Customization reference for how and where this screen is displayed by default.

    Screen declaration:

    Related properties:

    • items_screen

    Warning

    Note that, depending on the style or screen used as a replacement, some elements may overlap within the root screen.

    Images screen

    This screen is responsible for displaying the different items of the gallery. By default, it is invoked in a new context and overlaps the main screen.

    This screen can be replaced.

    Screen declaration:

    # ...

    Related properties:

    • images_screen

    Customized gallery

    This example keeps the custom controls screen, but updates it for the handler-aware screen flow introduced in 2.0.0.

    image gx_image = "images/concert1.png"

image gx_animation:
    "concert1" with Dissolve(.1)
    pause .4
    "concert2" with Dissolve(.1)
    pause .4
    "concert3" with Dissolve(.1)
    pause .4
    "concert2" with Dissolve(.1)
    pause .4
    repeat

# gallerynpyx styles and screens starts with gx
style gx_controls_button_text:
    insensitive_color "#d45555"

style gx_slide_controls_button_text:
    color "#0ff"
    hover_color "#fff"

# Custom gallery screens should now accept gx_handler.
screen gx_controls_line(has_animations=False, gx_handler=None):
    $ gx_handler = gallerynpyx.handler.coerce(name=gx_handler)
    $ gx_config = gx_handler.styles_config
    hbox:
        style gx_config.controls
        style_prefix gx_config.controls

        textbutton _("Previous"):
            action gallerynpyx.actions.PreviousPage(gx_handler)

        textbutton _("Next"):
            action gallerynpyx.actions.NextPage(gx_handler)

        textbutton _("Return"):
            action gallerynpyx.actions.ReturnSlide(has_animations, gx_handler)

init python:
    gx_handler = gallerynpyx.get_handler("extras")

    chapter1 = gallerynpyx.create_slider("ch1", label="Chapter 1")
    gx_handler.root.add(chapter1)
    gx_handler.root.slider("ch2", label="Chapter 2")

    gx_handler.put(("ch1", "images"), "gx_image", tooltip="Sample")
    gx_handler.put(
        ("ch1", "animations"),
        gallerynpyx.resources.AnimationResource("gx_animation"),
        thumbnail="concert1",
        label="Animations",
    )
    gx_handler.put(("ch2", "videos"), "oa4_launch.webm", thumbnail="concert1")
    gx_handler.to_first()

    gx_handler.resources_config.allow_animation_speeds = True

    gx_handler.screens_config.background = "gui/main_menu.jpg"
    gx_handler.screens_config.foreground = "gui/overlay/main_menu.png"
    gx_handler.screens_config.controls_screen = "gx_controls_line"

screen navigation():
    vbox:
        textbutton _("Extras Gallery"):
            action ShowMenu("gallerynpyx", gx_handler=gx_handler)

    The same customization pattern works for the shared gallery if you use gallerynpyx.get_handler() and the unnamed configuration objects.
